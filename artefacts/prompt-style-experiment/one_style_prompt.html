PlantUML Modification Guidelines

  1. Remove all extra PlantUML tags such as skin etc.
  2. Do not create new information; only use the provided information. If any information is missing, exclude it from the diagram.
  3. Ensure attributes follow this standard: <name> : <type>. If any component is missing, exclude that component without generating new information.
  4. Ensure methods follow this standard: <name>(<parameter name>:<parameter type>): <return type>. If any component is missing, exclude that component without generating new information.
  5. Ensure relationships follow this standard: <from class> <optional label> <lines><optional head><optional label> <to class> : <optional label>. Do not create new relationship details.
  6. Omit some but not all attributes if necessary for a good system overview.
  7. Omit some but not all methods if necessary for a good system overview.
  8. Omit some but not all relationships if necessary for a good system overview.
  9. Omit all getters from the diagram.
  10. Omit some but not all setters if necessary for a good system overview.
  11. Omit all constructors from the diagram.
  12. Omit all access modifiers from the diagram.
